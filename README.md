# 24ADI003_24BAD040_EXP4
Scenario 1:Multinomial Naive Bayes
Dataset:https://www.kaggle.com/datasets/uciml/sms-spam-collection-dataset
In this scenario, SMS messages are classified as Spam or Ham using the SMS Spam Collection dataset available on Kaggle. The text data is preprocessed through lowercase conversion, punctuation removal, and optional stopword removal. The cleaned messages are converted into numerical features using Count Vectorization or TF-IDF. A Multinomial Naïve Bayes classifier is trained on the processed data and evaluated using accuracy, precision, recall, and F1-score. Misclassified messages are analyzed to understand model limitations, and Laplace smoothing is applied to handle zero probabilities. Visualizations such as the confusion matrix, top spam-indicating words, and word frequency comparisons between spam and ham help interpret the model’s performance.
Scenario 2:Gaussian Naive Bayes
Dataset:Iris Dataset
In this scenario, the goal is to classify flower species using the Iris dataset based on features like sepal length, sepal width, petal length, and petal width. After preprocessing and feature scaling, the dataset is split into training and testing sets. A Gaussian Naïve Bayes classifier is trained and evaluated using accuracy, precision, recall, and F1-score. Predicted labels are compared with actual labels, and class probabilities are analyzed to understand confidence levels. Visualizations including decision boundary plots, confusion matrix, and probability distribution plots help illustrate classification performance and model behavior.
